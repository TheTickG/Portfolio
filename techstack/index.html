Tech Stack Summary

Diagram Overview

The tech stack diagram on the right visually represents the workflow of how I created my website and the various technologies involved in delivering the content. By choosing AWS Cloud, I leveraged its scalable and reliable infrastructure to ensure optimal performance and availability for my website.

Workflow Description

  1. User Interaction
    The process starts with the user accessing the website.

  2. Device and Browser
    The user interacts with the website through a Computer, which uses a Website Browser to request content.

  3. Domain Resolution - Route 53
    For domain resolution, I use AWS Route 53. Route 53 handles DNS resolution, directing the request to the appropriate content delivery network (CDN).

  4. Content Delivery - CloudFront
    I set up CloudFront to act as the CDN, caching and serving content. It optimizes content delivery and forwards requests to the backend server when necessary.

  5. Web Server - Amazon Linux EC2
    The request is ultimately served by the Amazon Linux EC2 Web Server, which runs the Apache web server. I use it to process the HTML, CSS, and JavaScript and deliver the web page to the browser. To keep within the AWS free tier, I chose a specific EC2 server that met the requirement.

Technologies Used

  • Route 53
    I chose AWS's Route 53 for domain name resolution, directing user requests to the correct resources.

  • CloudFront
    AWS's CloudFront CDN was selected to cache and deliver content quickly, reducing latency and improving load times.

  • Amazon Linux EC2
    For the web server, I utilized a free-tier version of Amazon Linux EC2 to stay within budget.

  • Apache Web Server
    To handle HTTP requests and serve web pages, I chose Apache Web Server for its reliability and extensive community support.

  • HTML
    I used HTML to structure the web content and provide the basic framework of the pages.

  • CSS
    I applied CSS to create the visual design and layout of the website.

  • JavaScript
    I included JavaScript to add interactivity and enhance the user experience on the web pages.

  • Mermaid
    This tool was used to create diagrams and visualizations in code. The diagram on the right, showing the tech stack flow, was made using Mermaid Markdown.

  • Git
    I used Git for version control to track changes in my codebase, manage updates, and collaborate on development.
